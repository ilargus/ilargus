# ilArgus Trademark Policy

This policy explains how the ilArgus name, logo, and branding may be used.

This policy does not change the Apache License, Version 2.0. The source code is licensed under Apache-2.0. The ilArgus name, logo, and branding are not licensed under Apache-2.0.

## Marks covered

The protected marks include:

- ilArgus
- The ilArgus logo
- The ilArgus icon, visual identity, and related branding
- Confusingly similar names or branding used for software, hosted services, packages, domains, or commercial products related to ilArgus

## Allowed uses without permission

You may use the ilArgus name without asking for permission when the use is truthful, non-confusing, and does not imply endorsement.

Examples of allowed use:

- Referring to the original ilArgus project
- Saying that your software is compatible with ilArgus
- Saying that your fork is based on ilArgus
- Linking to the official ilArgus repository
- Keeping copyright, license, and NOTICE references required by Apache-2.0

## Uses that require permission

You need written permission before you:

- Use the ilArgus name as the main name of a modified distribution
- Use the ilArgus name as the main name of a hosted service
- Use the ilArgus logo or icon in a product, website, app, package, or commercial offering
- Register or use a domain, social handle, package name, or app name that suggests official ilArgus status
- Use ilArgus branding in a way that implies endorsement, partnership, certification, or official status
- Use confusingly similar branding for infrastructure, monitoring, remote access, tunneling, or machine management software

## Forks and modified versions

You may fork and modify the ilArgus source code under Apache-2.0.

However, modified versions should use a clearly different name and branding. You may describe the relationship truthfully, for example:

- "Based on ilArgus"
- "A fork of ilArgus"
- "Compatible with ilArgus"

You may not present a modified version as the official ilArgus project unless you have written permission.

## Official project locations

The official ilArgus source repository is maintained under the ilargus GitHub organization.

Official project locations may include:

- The official ilArgus GitHub organization
- The official ilArgus website
- Official ilArgus package registries and container images
- Official ilArgus social accounts

If there is doubt about whether something is official, check the official repository first.

## Examples

Allowed:

- "MyTool is compatible with ilArgus."
- "This project is a fork of ilArgus."
- "This package integrates with ilArgus."

Not allowed without permission:

- Naming your hosted product "ilArgus Cloud"
- Publishing a modified binary called "ilArgus"
- Using the ilArgus logo for your fork
- Registering a domain that suggests it is the official ilArgus project
- Using branding that makes users think your project is maintained or endorsed by ilArgus
